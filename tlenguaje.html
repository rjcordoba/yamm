symbol-var puede ser una letra griega o su nombre, p. ej. α o alfa. Si se escribe el nombre se traduce a su símbolo.

mconfig-var es representado por una única letra mayúscula.

La llave de apertura no debe preceder nada en la misma línea. Si ocupa una línea por si misma no puede tener espacios antes.

La llave de cierre debe ocupar una línea por si misma, sin espacios antes.

Símbolos

Puede ser cualquier gráfico individual que se pueda representar con un codepoint unicode; en el caso de las letras griegas se puede usar su nombre.
Además se pueden usar los siguientes strings:

Any
Cualquier símbolo leído por la máquina.
None
Ningún símbolo.
...
Cualquier símbolo o no símbolo.
not s
Cualquier símbolo menos s
not s1 nor s2
Cualquier símbolo menos s1 o s2
::
Se considera un solo símbolo.
sw
Se sustituye por el símbolo ə.
0 o 1 precedido por -
El número con una barra encima.

Si se usa una letra griega antes de behaviour que no aparezca como variable en la m-function, esa letra representa cualquier simbolo que se lea (ejemplo abajo).

Operaciones

Si no se indica ninguna operación se pasa a la siguiente m-configuration sin hacer nada. Si hay, el significado es el siguiente:

R
La máquina pasa a leer el símbolo de la derecha del que lee actualmente.
L
La máquina pasa a leer el símbolo de la izquierda del que lee actualmente.
E
Se borra el símbolo de la posición leída.
Ps
Se imprime en la posición actual el símbolo s.

Nombres m-configurations

Los nombres deben estar formados por caracteres del alfabeto español, empezanado por una letra minúscula. Adicionalmente pueden acabar con cualquier número de comillas simples, seguidas a su vez de cualquier número de dígitos.

Ejemplos

  • b    None    P0, R    c

    En la m-configuration si no hay ningún símbolo en la posición actual se imprime «0» y se pasa a leer la posición de la derecha de la actual; la siguiente m-configuration será «c».

  • b
{
    None    P0       b
    0    R, R, P1    b
    1    R, R, P0    b
}

    En la m-configuration b:
    -Si no hay símbolo en la posición actual se imprime «1» y pasa a la m-configuration b.
    -Si se lee «0» desplaza el lector dos posiciones a la derecha, imprime «1» y pasa a la m-configuration b.
    -Si se lee «1» desplaza el lector dos posiciones a la derecha, imprime «0» y pasa a la m-configuration b. 

  • f(C, B, alfa){
    sw    L    f1(C, B, alfa)
    not alfa    L    f(C, B, alfa)
}

    Definición de skeleton.

  •  c1(C)    beta    pe(C, beta)

    Error
    La m-configuration inicial no debe estar precedida por ningún espacio.

  • c1(C)    beta    pe(C, beta)

    Ejemplo anterior corregido.
    En la m-configuration c1(C) lee cualquier símbolo y pasa a la mconfiguration pe(C, beta), donde beta es cualquier símbolo leído en el momento de cambiar de m-configuration.

 {
gamma    C
not gamma    A
}

    Error
    La llave de apertura que ocupa una línea no debe tener espacios antes.
    Las líneas que comienzan con un símbolo deben empezar con algún espacio.

  • cp2(C, A, gamma)
{
    gamma    C
    not gamma    A
}

    Ejemplo anterior corregido.

  • cr(B, alfa)    cr(cr(B, alfa), re(B, a, alfa), alfa)

    Error
    Debe haber algún símbolo.

  • cr(B, alfa)    ...    cr(cr(B, alfa), re(B, a, alfa), alfa)

    Ejemplo anterior corregido.
    Pasa de una m-configuration a otra con cualquier símbolo y sin símbolo.
